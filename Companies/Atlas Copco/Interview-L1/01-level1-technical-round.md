## Your Intro (2–3 min pitch)

**Template**
- Present: role, years, full-stack path (React + Node.js, learning backend).
- Core strengths: React for UIs, API integration, code quality, guiding others; actively expanding backend skills.
- Proof: 1 quantified impact (performance, delivery, or team improvement).
- What you want next: why Atlas Copco / industrial software / becoming well-rounded full-stack / willingness to learn Python.

**Suggested script (adapt to your experience)**
“I’m a Frontend Developer with [X] years of experience in React, and I'm actively expanding into full-stack — I'm on a path to learn Node.js for backend. I build high-performance, user-friendly frontends and understand API design, integration, and end-to-end flows from my work at the frontend–backend boundary.

I'm committed to becoming a well-rounded full-stack developer. This role fits that path perfectly. I'm keen to learn Python; the transition from JavaScript/Node to Python is well within reach given my fundamentals in APIs, async patterns, and backend thinking. I’m comfortable with Git, Docker, and CI/CD pipelines, and I follow security best practices like OAuth and JWT.

I’m excited about Atlas Copco’s work in industrial software and IoT. I’m looking for a role where I can build robust UI/HMI interfaces, contribute to APIs and integration, and grow into a stronger full-stack profile while adding Python to my toolkit.”

## Challenges (Pick 2)

**Template (5–7 min each)**
- Problem: what was broken / what was needed.
- Context: constraints (time, legacy, scale, stakeholders).
- Your ownership: what you personally drove.
- Design: options considered + tradeoff.
- Execution: milestones, how you shipped.
- Results: metrics or user/business impact.
- Reflection: what you’d improve next time.

**Ideas aligned to Atlas Copco JD**
1. **Full-stack feature**: Built a feature end-to-end (React + API); handled loading, errors, and performance.
2. **API design / integration**: Designed or worked closely with RESTful APIs (Node.js or any backend); made them reusable, reliable, secure. *If asked about Python: "I'm expanding backend skills with Node.js; the concepts transfer directly — I'm excited to apply them in Python."*
3. **Frontend–backend integration**: Ensured seamless communication between frontend and backend; handled edge cases and protocols.
4. **Embedded / resource-constrained**: Built or optimized web UIs for low-resource or embedded environments.
5. **Code quality / reviews**: Improved code quality or introduced code reviews; impact on reliability or velocity.
6. **Debugging / complex issue**: Solved a hard production bug or performance issue; your process (repro, isolate, fix, validate).
7. **Security**: Implemented OAuth, JWT, or similar auth; how you ensured secure API and frontend flows.

## Technical Depth (JD-aligned)

**React prep topics:** See `topics/00-index.md` for full coverage. Follow the preparation order for best results.

### React (Frontend)
- **Advanced React**: Component design, hooks, state management (Context, Redux), performance (memoization, code splitting).
- **High-performance UIs**: How you build responsive, performant user interfaces.
- **API integration**: Handling loading, errors, retries; keeping the app performant and resilient.

### If asked: "Do you know Python?" / "What about Python?"

**Honest, confident response:**
"I don't have production Python experience yet — I'm building my backend skills with Node.js first, which fits my path to become a well-rounded full-stack developer. The mental model is the same: RESTful APIs, async patterns, request handling, validation. I'm excited to learn Python on the job; the fundamentals transfer, and I learn quickly. This role would be a great place to add Python to my toolkit."

*Don't over-apologize. Emphasize: transferable skills, learning path, enthusiasm, fit with the role.*

### Backend (Node.js + Python willingness)
- **Your path (Node.js):** RESTful APIs, Express/Fastify, async patterns, request validation, error handling. *Lead with this — it's real and transferable.*
- **Python willingness:** "I'm actively building backend skills with Node.js. Python is my next target — the mental model (APIs, async, data flow) is the same. I learn quickly and I'm excited to contribute in Python once I'm productive. This role would accelerate that."
- **API design (language-agnostic):** RESTful design, reusable, reliable APIs; scalability; how backend services communicate with frontends and external systems.

### Embedded & Industrial (Awareness)
- **Embedded web**: Familiarity with web apps on embedded hardware (constraints, performance).
- **ModbusTCP**: Industrial protocol for connecting devices; basic awareness of use cases.
- **MQTT**: Lightweight messaging for IoT; pub/sub, brokers.
- *(If no direct experience: focus on transferable skills — APIs, protocols, integration, constraints.)*

### DevOps
- **Git**: Branching, PR workflow, merge strategies, code review.
- **Docker**: Containerization, images, volumes; why use Docker.
- **Azure DevOps**: Pipelines, boards, repos; familiarity with CI/CD concepts.
- **CI/CD**: Build, test, deploy automation; quality gates.

### Security
- **OAuth**: Authorization framework; flows, tokens, scopes.
- **JWT**: Structure, signing, validation; where to store tokens (frontend).
- **Application security**: Input validation, XSS, CSRF, secure headers.
