## Negotiation Plan (Datastack — Senior React Developer)

### Non-negotiable (acceptance criteria)
- **Fixed/Base**: Set your minimum fixed (base) — e.g. ₹**X–Y LPA fixed** → OK to accept.

### Preference order (when bridging gaps)
1. **Sign-on bonus** (preferred): certain + immediate.
2. **Variable pay** (fallback): only with clear payout rules and exact breakup in writing.

### Counter rule (if fixed is reduced)
- If they reduce fixed by **₹X**, ask for **₹X** additionally as **sign-on bonus** (first preference).
- If sign-on isn’t possible, ask for **₹X** additionally in **variable** (and confirm payout criteria + schedule).

### HR message template (copy/paste)
“To align on compensation: I’m good to proceed if the **fixed (base)** is in the range we discussed. If there’s any gap in fixed, my preference is to bridge it via a **sign-on bonus**. If sign-on isn’t feasible, we can discuss variable, but I’d want the payout terms clearly defined. Please share the **written breakup** (fixed, variable, bonus policy, PF/gratuity, and sign-on if applicable).”

### Must-ask clarifiers
- “Can you confirm **fixed/base** vs **variable** split explicitly in the offer?”
- “Is the sign-on **paid on joining** (or within first payroll), and what’s the **clawback** period?”
