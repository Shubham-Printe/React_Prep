# Level 1 Round — Prep (Datastack — Senior React Developer)

Use this as your script + checklist for the first interview. Keep answers crisp and structured.

---
## Coverage Checklist
- [ ] Your intro (2–3 min pitch) — React, state management, reusable components, leadership
- [ ] Challenge story 1 — delivery/ownership (e.g. feature, refactor, or rescue)
- [ ] Challenge story 2 — debugging / complex issue / performance or quality
- [ ] Role fit (why you / why this role — guidance to React team, best practices)
- [ ] Technical depth: React workflows (Flux/Redux), state management, APIs
- [ ] Code quality & reviews: how you write clean code and conduct reviews
- [ ] Leadership & guidance: mentoring, defining best practices, raising bar
- [ ] Questions to ask the interviewer

---
## Your Intro (2–3 min pitch)

**Template**
- Present: role, years, what you build (React, state management, reusable components).
- Core strengths: React/Redux (or Context), component design, API integration, code quality, guiding others.
- Proof: 1 quantified impact (e.g. performance, delivery, or team improvement).
- What you want next: why Datastack / why this role (ownership, leading React practices).

**Suggested script (adapt to your experience)**
“I’m a Senior React Developer with [X] years of experience building user interfaces with React and its ecosystem. I work with React workflows like Redux and Context API, design reusable components and front-end libraries, and integrate front-ends with back-end APIs for high performance.

I’m strong in state management, clean maintainable code, and code reviews. I also enjoy providing technical guidance to other React developers and defining best practices for the team.

I’m excited about Datastack because [research the company and add 1–2 lines]. I’m looking for a role where I can own UI quality, drive standards, and grow toward leading the React developer team.”

---
## Challenges (Pick 2)

**Template (5–7 min each)**
- Problem: what was broken / what was needed.
- Context: constraints (time, legacy, scale, stakeholders).
- Your ownership: what you personally drove.
- Design: options considered + tradeoff.
- Execution: milestones, how you shipped.
- Results: metrics or user/business impact.
- Reflection: what you’d improve next time.

**Ideas aligned to JD**
1. **Reusable components / front-end library**: e.g. built or refactored a design system or shared component set; how you made it reusable and maintainable.
2. **State management**: e.g. introduced or simplified Redux/Context; how you ensured data flow and reduced bugs.
3. **API integration / performance**: e.g. integrated a critical feature with back-end APIs; how you handled loading, errors, retries, and performance.
4. **Code quality / reviews**: e.g. improved code quality or introduced/enforced code reviews; impact on reliability or velocity.
5. **Debugging / complex issue**: e.g. solved a hard production bug or performance issue; your process (repro, isolate, fix, validate).

## Role Fit (Why you / why this role)

**Why you?**
- Strong fit: React + Redux/Context, reusable components, state management, API integration.
- Code quality: clean code, code reviews, adherence to standards.
- Leadership: experience guiding other React developers, defining best practices, aspiration to lead the React team.

**Why this role?**
- Want to own UI development and quality, provide technical leadership, and grow toward leading the React developer team.
- Datastack’s [product/domain — fill after research] aligns with where you want to have impact.

---
## Technical Depth (JD-aligned)

- **React workflows**: Be ready to explain when you use Flux/Redux vs Context; give a concrete example.
- **Reusable components**: Describe how you design components and libraries for reuse (API, variants, documentation).
- **State management**: Redux vs MobX vs Context — when and why; how you keep data flow clear.
- **API integration**: How you integrate with back-end services, handle errors/retries, and keep the app performant.
- **Build/tooling**: Familiarity with Webpack, Babel, npm/yarn (you don’t need to go deep in L1, but be able to mention usage).

---
## Code Quality & Reviews

- **Clean, maintainable code**: Naming, structure, separation of concerns, consistency.
- **Code reviews**: What you look for (correctness, performance, security, readability, tests); how you give feedback (clear, constructive).
- **Standards**: How you define or enforce coding standards (lint, formatter, PR checklist, docs).

---
## Leadership & Guidance

- **Guiding other React developers**: Examples of helping juniors or peers (pairing, reviews, documentation, tech talks).
- **Best practices**: How you define and spread best practices (e.g. patterns, performance, testing).
- **Aspiration to lead**: How you see yourself growing into leading the React developer team (ownership, standards, mentoring).

---
## Questions to Ask the Interviewer

- “What does success look like for this role in the first 30/60/90 days?”
- “How is the React team structured, and how do senior developers guide others?”
- “What are the biggest technical challenges in the frontend today?”
- “How do you balance code quality and delivery pressure?”
- “How are code reviews and coding standards handled?”
- “What’s the roadmap for front-end architecture, performance, or tooling?”
