## Phases at a glance

| Phase | Title |
|-------|--------|
| 0 | Prerequisites & decisions |
| 1 | Bootstrap (Expo + TypeScript + iOS simulator) |
| 2 | Project structure & conventions |
| 3 | UI foundation (lists, forms, feedback) |
| 4 | Navigation |
| 5 | REST API layer (client, types, env) |
| 6 | Server state (TanStack Query) & local UI state |
| 7 | Authentication & secure storage |
| 8 | Error handling, empty states, pull-to-refresh |
| 9 | iOS polish (safe area, keyboard, accessibility) |
| 10 | Build, TestFlight & App Store |
| 11 | Hardening (optional: tests, analytics, crash reporting) |

Read the roadmap in order; phases 5–7 are the core for **REST API** apps.
